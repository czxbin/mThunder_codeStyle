建议选项

  • bitwise: true
    禁止位运算，JS并不擅长为运算，大多时候出现位运算符其实都是逻辑运算符的缺失，如 && 写成 &。

  • camelcase: true
    只能使用驼峰命名或者 NAME_LIKE_THIS

  • curly: true
    大括号包裹

      //pass
  while (day) {
      shuffle();
  }
  //no pass
  while (day)
      shuffle();

  • eqeqeq: true
    不使用 == 或 != ，使用 === 或 !==

  • indent: 4
    缩进4个空格

  • latedef: true
    在变量使用前，必须先定义

  • newcap: true
    必须使用 new 来实例一个构造函数

  • noempty: true
    代码块不能为空

  • undef: true
    禁止使用未定义变量

  • strict: true
    启用严格模式

  • trailing: true
    禁止尾随空格

  • maxparams: 4
    设置最大参数个数为4

  • maxlen: 80
    一行最大的代码数为80

  • evil: false
    禁止使用eval

更多选项及内容，可浏览 JSHint docs

如何使用

  • 网站直接检测
    JSHint

  • 通过 nodeJS 使用 

      npm install jshint -g

  • gruntjs 的 JSHint 插件 （推荐）
    grunt-contrib-jshint

  • IDE 集成，编辑器插件 （推荐）

    • WebStorm 内部集成
    • Sublime Text
      sublime-jshint

更多使用方法，可浏览 JSHint install
